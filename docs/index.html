/pages/page.js

import { createState } from "elegance-js/helpers/createState";
import { observe } from "elegance-js/helpers/observe"; 

export const serverState = createState({
    counter: 0,
})

export const page = body ({
    // Set any attribute you want.
    class: "bg-black text-white",
},
    p ({ 
        // No black-boxes, just the observer pattern.
        innerText: observe({
            // Types and values inferred!
            [serverState.counter],
            // Whenever a value in the above id list changes, 
            // this function gets called with the new values. (fully type-safe!) 
            (counterValue) => `${counterValue}`,
        })
    }),

    button ({
        onClick: eventListener((state: State) => {
            const counter = state.subjects.counter;

            state.set(counter, counter.value + 1);

            // Explicit state signalling!
            state.signal(counter)
        })
    })
)

Key Features

Simple by Design

Elegance is not a black box with thousands of moving parts, nor does it have 10 years of tech-debt.
Everything is made in-house using as few depencencies as possible (1), with modern vanilla typescript.

By learning Elegance, you know exactly how it works, and what it does.
This in-depth knowledge allows you to utilize Elegance to it's fullest, allowing you to create the best user experience.

Fully Reactive

Things you've come to learn, or even love from React like functional components, conditional rendering, client-side navigation, etc. Are all fully available in Elegance; no extra packages needed.

Strict & Opinionated

Developers are lazy. Which is not as bad as it sounds. However, it does mean that if you give them 10 ways of doing something, they will choose the way that is quickest, easiest and most comfortable to them.

This often results in sub-optimal code. It's impacts may not always be apparent, and sometimes may not show at all for a while, however when put to scale, small issues become big issues.

By forcing developers to code correctly, you can prevent these issues from ever become ones in the first place.
Elegance Page Rater can be mounted on any page to address things like contrast, alt descriptors, component render time, etc.
