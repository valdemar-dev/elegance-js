State

Available Via: elegance-js/server/state

State is, simply put, a collection of variables.
You initialize it on the server using the createState() function.

<span class="text-amber-100 font-semibold">const</span> <span class="text-orange-300">superEpicState</span> = <span class="text-red-300">createState</span><span class="text-gray-400">(</span><span class="text-green-200">&quot;MMMMMMM STATE&quot;</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span>

Usage

The createState() function takes in two values.
The initial value of the state, and an options object.
The options object may currently only define a bind to the state (more on this later)

The function stores the created state in the servers current state store,
so that upon completion of compilation, it may be serialized into page_data.

Return Value

The return value of createState() is a State Object Attribute,
which you can use to refer back to the created state.

<span class="text-gray-400">{</span>    <span class="text-orange-300">type</span>: <span class="text-orange-300">ObjectAttributeType</span><span class="text-gray-400">.</span><span class="text-orange-300">STATE</span><span class="text-gray-400">,</span>    <span class="text-orange-300">id</span>: <span class="text-blue-400">0</span><span class="text-gray-400">,</span>    <span class="text-orange-300">value</span>: <span class="text-green-200">&quot;MMMMMMM STATE&quot;</span><span class="text-gray-400">,</span>     <span class="text-orange-300">bind</span>: <span class="text-orange-300">undefined</span><span class="text-gray-400">,</span><span class="text-gray-400">}</span>
<span class="text-amber-100 font-semibold">const</span> <span class="text-orange-300">isUsingDarkMode</span> = <span class="text-red-300">createState</span><span class="text-gray-400">(</span><span class="text-blue-200">false</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span><span class="text-red-300">div</span> <span class="text-gray-400">(</span><span class="text-gray-400">{</span>    <span class="text-amber-100 font-semibold">class</span>: <span class="text-red-300">observe</span><span class="text-gray-400">(</span>        <span class="text-gray-400">[</span><span class="text-orange-300">isUsingDarkMode</span><span class="text-gray-400">]</span><span class="text-gray-400">,</span>        <span class="text-gray-400">(</span><span class="text-orange-300">value</span><span class="text-gray-400">)</span> => <span class="text-orange-300">value</span> ? <span class="text-green-200">&quot;bg-black&quot;</span> : <span class="text-green-200">&quot;bg-white&quot;</span><span class="text-gray-400">,</span>    <span class="text-gray-400">)</span><span class="text-gray-400">,</span><span class="text-gray-400">}</span><span class="text-gray-400">)</span>

Many functions like load hooks, event listeners, and observe, take in optional SOAs.

Bind

State, in the browser, is kept in the global pd object, and indexed via pathnames.
All state for the pathname /recipes/apple-pie will be in pd["/recipes/apple-pie"]

However, in some scenarios you may want to reference the same state on multiple pages.
For this, you may bind the state to a Layout.

Then, the state will go into pd[layout_id], instead of the pathname of the page.

<span class="text-amber-100 font-semibold">const</span> <span class="text-orange-300">docsLayout</span> = <span class="text-red-300">createLayout</span><span class="text-gray-400">(</span><span class="text-green-200">&quot;docs-layout&quot;</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span><span class="text-amber-100 font-semibold">const</span> <span class="text-orange-300">timeSpentOnPage</span> = <span class="text-red-300">createState</span><span class="text-gray-400">(</span><span class="text-blue-400">0</span><span class="text-gray-400">,</span> <span class="text-gray-400">{</span>    <span class="text-orange-300">bind</span>: <span class="text-orange-300">docsLayout</span><span class="text-gray-400">}</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span>

Important Considerations

State persists it's value during page navigation.
Meaning if you want it to reset it's value, you must do so yourself.

Load Hooks

Available Via: elegance-js/server/loadHook

Basic Usage

Load hooks are functions that are called on the initial page load, and subsequent navigations.
A load hook is registered using the createLoadHook() function.

<span class="text-red-300">createLoadHook</span><span class="text-gray-400">(</span><span class="text-gray-400">{</span>    <span class="text-orange-300">fn</span>: <span class="text-gray-400">(</span><span class="text-gray-400">)</span> => <span class="text-gray-400">{</span>        <span class="text-orange-300">console</span><span class="text-gray-400">.</span><span class="text-red-300">log</span><span class="text-gray-400">(</span><span class="text-green-200">&quot;The page has loaded!&quot;</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span>    <span class="text-gray-400">}</span><span class="text-gray-400">,</span><span class="text-gray-400">}</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span>

Cleanup Function

The return value of a load hook is referred to as a cleanup function.
It is called whenever the load hook goes out of scope.

You'll want to do things like clearInterval() & element.removeEventListener()
here, so you don't get any unintended/undefined behavior.

class="text-gray-400">}</span><span class="text-gray-400">,</span><span class="text-gray-400">}</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span>

Load Hook Scope

The scope of a load hook is either the page it is on, or the layout it is bound to.
If a load hook is bound to layout, it is called when that layout first appears.
Subsequently, its cleanup function will get called once it's bound layout no longer exists on the page.

To bind a load hook to a layout, use the bind attribute, and pass in a Layout ID

<span class="text-amber-100 font-semibold">const</span> <span class="text-orange-300">layout</span> = <span class="text-red-300">createLayout</span><span class="text-gray-400">(</span><span class="text-green-200">&quot;epic-layout&quot;</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span><span class="text-red-300">createLoadHook</span><span class="text-gray-400">(</span><span class="text-gray-400">{</span>    <span class="text-orange-300">bind</span>: <span class="text-orange-300">layout</span><span class="text-gray-400">,</span>    <span class="text-orange-300">fn</span>: <span class="text-gray-400">(</span><span class="text-gray-400">)</span> => <span class="text-gray-400">{</span>        <span class="text-red-300">alert</span><span class="text-gray-400">(</span><span class="text-green-200">&quot;epic layout was just rendered&quot;</span><span class="text-gray-400">)</span>        <span class="text-red-300">return</span> <span class="text-gray-400">(</span><span class="text-gray-400">)</span> => <span class="text-gray-400">{</span>            <span class="text-red-300">alert</span> <span class="text-gray-400">(</span><span class="text-green-200">&quot;epic layout is no longer with us :(&quot;</span><span class="text-gray-400">)</span>        <span class="text-gray-400">}</span><span class="text-gray-400">;</span>    <span class="text-gray-400">}</span><span class="text-gray-400">,</span><span class="text-gray-400">}</span><span class="text-gray-400">)</span>

Important Considerations

It's important to note that the load hook function body exists in
browser land not server land. Therefore the code is untrusted.

Event Listener

Available Via: elegance-js/server/createState

Basic Usage

Event listeners are a type of state, that you can create with the
createEventListener() function.

<span class="text-amber-100 font-semibold">const</span> <span class="text-orange-300">handleClick</span> = <span class="text-red-300">createEventListener</span><span class="text-gray-400">(</span><span class="text-gray-400">{</span>    <span class="text-orange-300">eventListener</span>: <span class="text-gray-400">(</span><span class="text-orange-300">params</span>: <span class="text-orange-300">SetEvent</span><<span class="text-orange-300">MouseEvent</span><span class="text-gray-400">,</span> <span class="text-orange-300">HTMLDivElement</span>><span class="text-gray-400">)</span> => <span class="text-gray-400">{</span>        <span class="text-orange-300">console</span><span class="text-gray-400">.</span><span class="text-red-300">log</span><span class="text-gray-400">(</span><span class="text-orange-300">params</span><span class="text-gray-400">.</span><span class="text-orange-300">event</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span>        <span class="text-orange-300">console</span><span class="text-gray-400">.</span><span class="text-red-300">log</span><span class="text-gray-400">(</span><span class="text-orange-300">params</span><span class="text-gray-400">.</span><span class="text-orange-300">event</span><span class="text-gray-400">.</span><span class="text-orange-300">currentTarget</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span>    <span class="text-gray-400">}</span><span class="text-gray-400">,</span><span class="text-gray-400">}</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span><span class="text-red-300">div</span> <span class="text-gray-400">(</span><span class="text-gray-400">{</span>    <span class="text-orange-300">onClick</span>: <span class="text-orange-300">handleClick</span><span class="text-gray-400">,</span><span class="text-gray-400">}</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span>

This function returns an SOA, which can then be put on any event listener option of an element.

The eventListener parameter of createEventListener() takes in two types values.
First, a params object, which by default contains the native event which was triggered.

Dependencies

The second parameter, is a spread parameter, containing the dependencies of the event listener.

Layouts

Available Via: elegance-js/server/layout

A layout is a section of a page that is not re-rendered between
page navigations, to pages that share the same layout order.

Instead, the layouts children are replaced.

This has a few advantages. The main one being, that since the elements themselves,
are not re-rendered, they maintain things like their hover state.

Basic Usage

Layouts work a bit differently in Elegance than you may perhaps be used to.
For example, in Next.JS, layouts are inherited to every subsequent page.

So a layout defined at / would apply to every single page.
Which you may think is nice and saves time, but almost always I find myself in a situation
where I want a layout for every page of a given depth, except one.

And then, I have to either move the special page one depth upward
or the others one-depth downward.

Conversly, layouts in Elegance are not inherited, and are are opt-in.

To create a layout, use the createLayout() function, and pass in a name.
The name is used so any subsequent calls to this function by other pages will get the same ID.

<span class="text-amber-100 font-semibold">const</span> <span class="text-orange-300">superAwesomeLayoutID</span>  = <span class="text-red-300">createLayout</span><span class="text-gray-400">(</span><span class="text-green-200">&quot;super-awesome-layout&quot;</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span>

This layout ID can then be passed to state, load hooks, etc.

Breakpoints

Creating the actual layout element is simple.
Just make a function that takes in child elements, and have it return some kind of simple layout.

<span class="text-amber-100 font-semibold">const</span> <span class="text-orange-300">superAwesomeLayoutID</span> = <span class="text-red-300">createLayout</span><span class="text-gray-400">(</span><span class="text-green-200">&quot;super-awesome-layout&quot;</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span><span class="text-amber-100 font-semibold">const</span> <span class="text-orange-300">SuperAwesomeLayout</span> = <span class="text-gray-400">(</span><span class="text-gray-400">.</span><span class="text-gray-400">.</span><span class="text-gray-400">.</span><span class="text-orange-300">children</span>: <span class="text-orange-300">Child</span><span class="text-gray-400">[</span><span class="text-gray-400">]</span><span class="text-gray-400">)</span> => <span class="text-red-300">div</span> <span class="text-gray-400">(</span><span class="text-gray-400">{</span>    <span class="text-orange-300">style</span>: <span class="text-green-200">&quot;background-color: #000; color: #fff&quot;</span><span class="text-gray-400">,</span><span class="text-gray-400">}</span><span class="text-gray-400">,</span>    <span class="text-gray-400">.</span><span class="text-gray-400">.</span><span class="text-gray-400">.</span><span class="text-orange-300">children</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span>

Then, wrap the children with the built-in Breakpoint() element.

<span class="text-amber-100 font-semibold">const</span> <span class="text-orange-300">superAwesomeLayoutID</span> = <span class="text-red-300">createLayout</span><span class="text-gray-400">(</span><span class="text-green-200">&quot;super-awesome-layout&quot;</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span><span class="text-amber-100 font-semibold">const</span> <span class="text-orange-300">SuperAwesomeLayout</span> = <span class="text-gray-400">(</span><span class="text-gray-400">.</span><span class="text-gray-400">.</span><span class="text-gray-400">.</span><span class="text-orange-300">children</span>: <span class="text-orange-300">Child</span><span class="text-gray-400">[</span><span class="text-gray-400">]</span><span class="text-gray-400">)</span> => <span class="text-red-300">div</span> <span class="text-gray-400">(</span><span class="text-gray-400">{</span>    <span class="text-orange-300">style</span>: <span class="text-green-200">&quot;background-color: #000; color: #fff&quot;</span><span class="text-gray-400">,</span><span class="text-gray-400">}</span><span class="text-gray-400">,</span>    <span class="text-red-300">Breakpoint</span> <span class="text-gray-400">(</span><span class="text-gray-400">{</span>        <span class="text-orange-300">id</span>: <span class="text-orange-300">superAwesomeLayoutID</span>    <span class="text-gray-400">}</span><span class="text-gray-400">,</span>        <span class="text-gray-400">.</span><span class="text-gray-400">.</span><span class="text-gray-400">.</span><span class="text-orange-300">children</span>    <span class="text-gray-400">)</span><span class="text-gray-400">,</span><span class="text-gray-400">)</span><span class="text-gray-400">;</span>

Important Considerations

The Breakpoint() element is the one that gets replaced
when navigating within any given layout.

All sibling and parent elements stay untouched.

Also note, that in complex pages where there are multiple nested layouts,
the one that has its children replaced, is the layout that is last shared.

For example:
Page 1 Layouts: A,B,C,D,E
Page 2 Layouts: A,B,D,C,E
In this instance, the B layout is the last shared layout.
